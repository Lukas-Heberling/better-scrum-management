Get started

    Client Response Format:
 
      {
        messages: {
          infos?: string[];
          warnings?: string[];
          errors?: string[];
        }
        values: any;
        type: ('success' | 'error');
      }

The PBC backend always responds to the client in the same data structure.
Dies ist die API Dokumentation des PBC (Product Backlog Creator) Backends.
Das PBC Backend händelt das anlegen, händeln, entfernen und laden von abteilungen, sprints und tickets.
The key "messages" is used to send messages to the client, which can be displayed in the frontend.
The key "values" can be anything and will contain the data that the client requested.
The key "type" is used to indicate that everthing worked fine or an error occured.

Get Sprints

If you want to retrieve sprints, you will retrieve all sprints for a specific department.
To get the sprints you will have to use the following url: http://[backend]/sprint/getAllFor/[departmentId] 



 Result example :
 
 {
    "messages": {
      "infos": [
        "Successfully selected all the rows for the departemnt"
      ]
    },
    "values": [
      {
        "sprint_id": 1,
        "sprint_number": 1,
        "sprint_name": "First Sprint",
        "department_id": 1
      },
      {
        "sprint_id": 2,
        "sprint_number": 2,
        "sprint_name": "Second Sprint",
        "department_id": 1
      }
    ],
    "type": "success"
  }

QUERY PARAMETERS

Field Type Description
department_id String The ID of the department you want to get the sprints from

Create Sprint

If you want to create a Sprint you will have to use the following URL: http://[backend]/sprint/create/[department_id]/[sprint_name]/[sprint_number] After the Sprint got created it will return the created sprint. 



 Result example :
 
 {
  "messages": {
    "infos": [
      "Successfully created a new Sprint"
    ]
  },
  "values": [
    {
      "sprint_id": 4,
      "sprint_number": 1,
      "sprint_name": "FirstSprint",
      "department_id": 1
    }
  ],
  "type": "success"
}

QUERY PARAMETERS

Field Type Description
department_id Number The ID of the department you want create a sprint for
sprint_name String The Name for the new Sprint
sprint_number Number The Number of the created Sprint
department_id Number The ID of the department you want to get the sprints from

Remove Sprint

If you want to remove a sprint from a department you can do that by the following URL:
http://[backend]/sprint/remove/[sprintId] 



 Result example :
 
 {
    "messages": {
      "infos": [
        "Successfully removed Sprint"
      ]
    },
    "values": [],
    "type": "success"
  }

QUERY PARAMETERS

Field Type Description
sprint_id Number The ID of the Sprint you want to remove
